KEY OBSERVATIONS / JUSTIFICATION OF BIG-O

The sorting algorithm applied to sort student CGPA is Introsort. Introsort is a hybrid of Quicksort, Heapsort and Insertion Sort. The time complexity is O(n log n) in all cases and only requires a small amount of space O(log n) to track steps and organise data due to the recursive call stack.

For searching and managing records, efficiency depends on the data handling method. Linear search checks students one by one from the start, with a worst case where every element must be visited — O(n) time. Time taken increases linearly with dataset size and no extra memory is needed.

For binary search, the array must be sorted before every search. Although binary search itself is O(log n), the required Introsort step costs O(n log n), making the total cost per call O(n log n). This is why binary search shows higher times than plain linear search in the table. It also uses O(log n) stack space from the Introsort recursive calls.

For adding and deleting records, the algorithm first uses linear search to locate the target O(n), then shifts all subsequent elements to create or close a gap O(n). Both operations are therefore O(n) overall. Displaying all records visits each record exactly once, giving O(n) time and O(1) space as no additional memory is needed beyond the loop variable.
